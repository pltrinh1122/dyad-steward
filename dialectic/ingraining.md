> **Status: DRAFT, n=1. Telos-critical.** Opened 2026-05-30 from a Steward Operator `[FEEDBACK]`:
> repeated Agent re-emission of a deprecated sibling-dyad name "undermines confidence in the
> **ingraining capacity** of our system — the core thesis underpinning my belief in 1+1=3." This is the

## The failure (n=1, grounded via `gh`)

The Agent re-emitted a deprecated repo name **three times across one session, twice *after* explicit
correction**. Root cause:

- **Single source:** the deprecated token survives in the form's `AGENT.md` (the DIP) — the most
  authoritative, most-read document in the practice — *inconsistent with its own adjacent URL and with
  the subject repo's self-name*. Reading the DIP this session put the dead token in-context from the
  highest-authority source.
- **Three mechanisms:**
  1. **Authoritative source re-emits** on every read (bail the boat, hole still open).
  2. **Correction amplifies** — re-typing the dead token in a `[FEEDBACK]`/ack raises its *in-context
     frequency*; the Agent interpolates from frequency, so *correcting it fed it*.
  3. **A false generative rule** (`dz-→dyad-` prefix swap) substituting for grounding —
     *coincidentally* right for one case, false in general (`dz-cil → dyad-wu-wei`, **not**
     `dyad-cil`; the old `ta` name → no repo). The rule would have made the Agent *invent* a name.

## The lesson — ingraining ≠ Agent memory; it lives in reloaded substrate

**The Agent has no persistent across-session memory and is dominated within-session by in-context
frequency.** So "correct the Agent and it will remember" is **false** — and this failure is the proof.

In *this* system, therefore:

> **Ingraining = durable encoding in the substrate that reloads** (the anchor, `kb/`, `dialectic/`,
> memory) **+ keeping the Agent's *sources* correct.** Not a habit in the Agent's head — across
> sessions, the Agent has no head.

**Mirror of `dyad-ui.md` §Markers** ("a marker vocabulary ingrains in the Operator's *wet-substrate*;
you can't legislate a habit"): the Agent's mirror is that it has *no* wet substrate to legislate — its
learning **must** be externalized. Same principle, both halves: **ingraining lives in the record, not
the head** — the Operator's memory is *lossy*, the Agent's is *absent*; the durable record is neither.

### The discipline (corollaries)

1. **Fix the source, not the output.** A wrong authoritative source re-seeds the grain every read.
2. **Correcting in-context amplifies — so stop quoting the dead token** once located; refer to it
   indirectly. (Negation raises salience; "don't think of X" keeps X maximally present.)
3. **Ground per-instance; never extrapolate a rule.** `gh`/read is the debiasing move; a plausible
   rule is a confident hallucination waiting to fire.

## Why this *strengthens* 1+1=3 (correctly located)

The compounding "3" is **stored in the externalized, falsified, reloaded substrate** — which is
*exactly why the dyad beats either half over time*: not the Operator's lossy memory, not the Agent's
absent memory, but the durable record both build and reload. The thesis was never "the Agent learns
like a human"; it is "**the dyad externalizes survived learning into substrate, and that substrate
compounds.**" The failure falsifies the *wrong model* (Agent-recall), **not** the thesis.

**Demonstrated this very session:** when the correction triggered *grounding* (not exhortation), the
system caught mechanism (3) and **avoided inventing `dyad-cil`**. The empirical-debiasing half worked
in real time — 1+1=3 firing. What failed was relying on Agent-recall, not the dyad.

## The honest threat (no reassurance theatre)

The lost confidence is **partly justified.** Our ingraining *practice* was sloppy — we left an
authoritative source wrong, corrected by exhortation, never codified the naming. **If ingraining is
not made as ruthless as falsification, the doubt is warranted.** The burden is on us; this cycle + its
fixes are the first instance of discharging it, and the Operator should hold us to the test below.

## Falsifiable claim

Ingraining located in *reloaded substrate + corrected sources* **sticks**; ingraining attempted via
Agent-*exhortation* **does not**. *Test:* after the source-fix + this codification, the deprecated name
does **not** recur in a fresh session; future deprecated-grain cases are caught by *grounding*, not
repetition.

**Refuted if:** the name recurs *even after* the source is fixed and the lesson sits in the reloaded
anchor/memory (→ the substrate-location theory is wrong; deeper problem) · or grounding-per-instance
proves too costly to sustain (→ need a cheaper mechanism).
