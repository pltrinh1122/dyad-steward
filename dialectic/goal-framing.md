## The candidate — what `[GOAL]` opens

`[GOAL]` opens a **session-planning intake**: the Operator declares a *laundry list of intended
outcomes* — **conjunctive** (all to be achieved), not alternatives — usually right after a RESTART.
The Agent's job is **neither to generate** the goals (that would be `IFD`) **nor to dispose** them
(that would be `DFD`) but to **organize, type, Telos-vet, and sequence** them into a ratifiable
execution DAG. Out of the region: a **plan + execution DAG, awaiting Operator ratification before any
execution**.

> Where `DFD` moves the Operator authoring→**validating**, and `IFD` moves an exploration
> prompt→**survivor candidate**, **`GFD` moves a raw intent-dump → a Telos-vetted, dependency-ordered
> plan.**

**The load-bearing survivor — the Telos-gate.** Nothing in existing machinery vets *whether a stated
outcome should be pursued at all*: DFD disposes a decision; NBA orders eligible nodes; IFD falsifies
candidate *viability*. The `GFD` intake-gate falsifies each item against the **Telos** — *does this
outcome compound the commons' 1+1=3, or is it scope-creep / busywork / a private-fork temptation /
mis-scoped for this dyad?* This is the **NON-NEGOTIABLE (anti-sycophancy) applied at the intake
altitude**: the Agent's empirical-debiasing half testing the **Operator's premises about what to do**,
as rigorously as it tests Agent proposals (`kb/one-plus-one-equals-three.md`). Items that fail are
**surfaced with evidence — never silently dropped, never silently honored.**

## The flow — `GFD`

1. **Intake.** Operator declares `[GOAL]` + the list (usually post-RESTART). Agent receives all items;
   deduplicates; **clarifies any ambiguous item before proceeding** (no silent interpretation).
2. **Telos-gate** *(the new primitive)*. For each item, a genuine falsification against the Telos →
   verdict ∈ {**serves** · **qualified** (serves but needs reshaping/narrowing) · **fails** (does not
   serve / mis-homed / out-of-scope)}, evidence folded in. Failures + qualifications populate the
   plan's **anti-thesis**. *The Operator may override a `fails` (their intent governs) — but only
   against the stated objection, never by default; the override is recorded (see Open questions).*
3. **Type + order.** Type each survivor into a work-node (this is the concrete **pull** on the
   types-of-work typology — `dyad-work.md` GAP) and assemble the **execution DAG** (NBA: dependencies
   → ready-set). Deferrals become honest (a node with an unmet edge); parallel fronts surface.
4. **Surface via `GF-UI`** (below), terminating in a **single** `[CTA·Y/N]` to **ratify the plan**.
5. **Gate.** **No execution before ratification.** The ratification-Y is a *plan-level sequencing*
   dispose; per `nba-dag.md`'s two-gate lesson, generative nodes inside the plan still owe their own
   **content-CTA** when their artifact exists.

## Falsifiable claim

Running a session's intent-dump through `GFD` (Telos-gate → typed DAG → one ratification) yields a
session that is **more Telos-aligned and better-sequenced** than executing the laundry list in its
declared order. *Test:* across GOAL-opened sessions, the Telos-gate **at least sometimes** reshapes or
rejects an item, **and** the DAG **at least sometimes** reorders vs. declared order.

**Refuted if:**
- **Gate theatre** — the Telos-gate passes every item unchanged across several sessions → it's a
  rubber-stamp; cut it to a one-line "all aligned" and drop the ceremony.
- **Sequence-degeneracy** — the DAG never reorders the declared list → the laundry list was already a
  plan; `GFD` adds nothing `NBA→DFD` didn't.
- **Over-naming fires** — `GF-UI`'s show/flow split never comes apart (the surface is *always* just
  IF-UI + DAG + CTA, no GOAL-specific behavior) → collapse `GF-UI`; keep only the Telos-gate as a
  pre-step to `NBA→DFD`.
- **Premature-structure / wu-wei breach** — this file, written at **n=0**, encodes a shape the first
  real GOALs contradict. *Mitigation:* it is **DRAFT**; the first real GOAL is seed evidence,
  **distilled-from-practice, not legislated** (the node-M lesson: *you can't legislate a habit*; IFD
  and DFD were each distilled from ~7 lived instances, not invented).
