## `[GOAL]` — what it opens

A **session-planning intake**: the Operator declares a **conjunctive** laundry list of intended
outcomes (all to be achieved, *not* alternatives), usually post-RESTART. The Agent **organizes,
Telos-vets, types, and sequences** them into a ratifiable execution DAG. **No execution before the
plan is ratified.** *(Neither generate the goals (→`IFD`) nor dispose them (→`DFD`): `GFD` turns a raw
intent-dump into a Telos-vetted, dependency-ordered plan.)*

## Flow — `GFD`

1. **Intake** — receive all items; **dedupe**; **clarify any ambiguous item before proceeding** (no
   silent interpretation).
2. **Telos-gate** — for each item, **falsify against the Telos** → verdict ∈ {**serves** ·
   **qualified** (needs reshaping/narrowing) · **fails** (mis-homed / out-of-scope)}, evidence folded
   in. **Criteria:** *does it compound the commons' 1+1=3, or is it scope-creep / busywork / a
   private-fork temptation / mis-scoped for this dyad?* Failures + qualifications populate the plan's
   **anti-thesis** — **surface with evidence; never silently drop or silently honor.** The Operator
   may override a `fails` only **against the stated objection** (never by default); **record the override.**
3. **Type + order** — type each survivor into a work-node; assemble the **NBA execution DAG**
   (dependencies → ready-set; honest deferrals = nodes with unmet edges; surface parallel fronts).
4. **Surface via `GF-UI`** — terminating in a **single `[CTA·Y/N]`** to **ratify the plan**.
5. **Gate** — **no execution before ratification.** Plan-Y is a *sequencing* dispose; generative nodes
   inside still owe their own **content-CTA** when their artifact exists (`nba-dag.md`).

## Show — `GF-UI`

**N Operator-provenance items** + **Telos-verdict column** (serves/qualified/fails + evidence) +
**NBA DAG-render** (ready-set · edges · deferrals) + **single terminal `[CTA·Y/N]`**. No new markers
beyond `[GOAL]`. **Decidable-in-flow** — reviewable and ratifiable without leaving the flow.
