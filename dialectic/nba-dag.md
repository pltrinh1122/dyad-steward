## The candidate — a composed loop, not a calculator

When we embark on a **multi-activity job**, set up a **DAG** of the activities and run this loop:

```
NBA (math, machine-safe)            DFD (dispose, Operator-only)
─────────────────────────           ────────────────────────────
DAG → topological frontier          Agent recommends one  →  [THESIS]
   = READY-SET (unblocked nodes)     runner-up ready nodes →  [ANTI-THESIS]
        │                            framed disposition    →  [CTA·Y/N]
        └──────────── hands off ────────────────► Operator disposes Y/N
```

- **NBA computes the *ready-set*** — the eligible frontier (nodes whose prerequisites are all met).
  Structural, automatable.
- **NBA never emits "best."** "Best among eligible" is a *value* judgment = the Operator's dispose.
  The Agent picks a **recommended** action from the ready-set — a **Generate +1**, so it lives in
  `[THESIS]`; the `[CTA·Y/N]` is exactly what keeps it a recommendation, not a verdict.

## Why Composition is the load-bearing move (not just tidiness)

The composition with DFD is the **structural fix** for the refuted claim:

## Loop mechanics — the two gates (sequencing ≠ content)

**The NBA→DFD CTA authorizes *which node* to work — that is a *sequencing* dispose, not a *content*
one.** A node whose output is a **settled artifact** (a discipline, a definition — DISPOSE-family
terminus) then owes a **second, content** dispose before it is truly complete:

| Node terminus | Gates owed |
|---|---|
| **DISPOSE-family** (produces a settled artifact) | **two** — sequencing-CTA (do it next?) **then** content-CTA (is the result right?) |
| **EXPLORE-family** (an exploration) | one — converges *open*, no dispose (IFD) |
| **mechanical** (action = content, e.g. "promote this vocab") | one — sequencing-Y *is* the content-Y |

The two are **sequential gates, not simultaneous asks** (so the single-CTA discipline holds): the
sequencing-CTA authorizes the work; the content-CTA fires *after* the artifact exists, rendered
decidable-in-flow. The cost (a second dispose on generative nodes) is **necessary** — skipping it is
exactly the bug below.

> **Lived (2026-05-30):** nodes **M** (marker taxonomy) and **I** (IFD) were *committed on their
> sequencing-Y's* — I treated "work on I next" as license to author **and** enshrine I. The Operator

## Falsifiable claim

The composed loop makes the next unit of a multi-activity job **cheaper and safer** than picking by
feel: no lost work on blocked nodes, parallel fronts made visible, and the Operator's dispose
*grounded* in a real ready-set + real alternatives. *Test:* across multi-activity jobs, NBA reorders
vs. pick-any-ready at least sometimes, and the runner-up nodes at least sometimes inform the dispose.

**Refuted if:**
- **Singleton degeneracy** — the ready-set is *always* one node → the DAG never reorders → collapse
  NBA back to "pick a ready one" (the math earned nothing).
- **Dead anti-thesis** — runner-up nodes never inform a dispose → the second payoff is fiction.
- **UI-inflation** (borrowed from `dyad-ui.md`) — naming the seam adds ceremony to a reflex
  (recommend → dispose) we already had, with no catch to show for it.
- **Premature-structure / wu-wei breach** — a frozen up-front DAG encodes edges the (emergent,
  dialectical) work later contradicts. Mitigation: keep the DAG **living** — re-derive as the
  dialectic produces nodes; build it only when N and edge-depth earn it.
