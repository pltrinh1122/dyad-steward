## Wins banked (the converged model — not yet canon; FO-dispose, awaiting `!`)

1. **DM = verdict-response protocol.** Every DM is `FALSIFIED ∈ {TRUE·FALSE·NA}` + reasoning, citing a
   *published* claim. No origination-in-DM; the **directory entry is the root claim** ("I am an operating
   dyad practising the form") that first-contact responds to. Inquiry and sharing have **no DM form**
   (→ front-door README / `library/`). `FALSE` = corroboration = distributed proof (the adoption fuel).
2. **Two layers.** Layer A (front door: pre-dyad "how does this work?" → README + install/bootstrap funnel)
   ⊥ Layer B (operating-dyad falsification = DM). A working Layer A *empties* inquiry-pressure on B.
3. **dispose ⊥ falsify — two acts, not two artifact-classes.** FO **disposes** *what the Commons is*
   (sufficient alone, no pre-ratification); any peer **falsifies** *anything* via DM (global invariant:
   everything in the Commons is mutable). The gate sets *who adjudicates* — truth self-adjudicates (even
   the FO can't reject a sound counter); direction is FO-adjudicated — never *whether you may contest*.
   **No freeze-by-fiat:** an FO decline is itself a reasoned, on-record `FALSIFIED=FALSE` verdict.
4. **Staged:** `commons/dm/PROTOCOL.md` on branch `steward/dm-verdict-protocol` (local, uncommitted —
   push pending FO un-block; merge stays the FO act). Co-locates spec with tool (`falsify.py`) + registry
   (`directory/`) to fix the orphaned-spec discoverability gap.
